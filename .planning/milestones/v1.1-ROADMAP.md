## Overview

Make GSD work survive context resets. Before v1.1, a `/compact` (or natural context exhaustion) would wipe the session's mental model: phase position, plan progress, task breadcrumbs, and any decisions made in the conversation. The user had to manually re-orient the assistant every time. v1.1 solves this with an automatic checkpoint-write-on-compact and auto-resume-on-session-start, backed by a CLAUDE.md fallback path for environments where hooks don't fire, and a cleanup step so stale handoff files don't trigger phantom resumes.

The milestone was originally scoped to 3 phases across checkpoint creation, lifecycle polish, and upstream-compat + documentation. Mid-milestone audit (AUDIT-v1.1.md) found upstream GSD had evolved in ways that invalidated Phase 6's compat target, and several Phase 5 requirements were speculative polish. Option C rescope (2026-04-20) dropped Phase 6 entirely and trimmed Phase 5 to the two genuinely missing pieces (CLAUDE.md fallback, HANDOFF.json cleanup). Shipped that scope later the same day.

## Key accomplishments

1. **End-to-end session continuity across `/compact`** — PreCompact hook writes HANDOFF.json (19-field checkpoint schema); SessionStart hook detects it and emits systemMessage; Claude auto-invokes `/gsd:resume-work`. Live round-trip verified 2026-04-20. Zero user intervention required.
2. **Hook-independent fallback path** — `## Session Continuity` section in CLAUDE.md provides a second trigger channel for CLIs without hook support or when the hook is overridden. Same skill executes; same end state.
3. **Full handoff lifecycle** — creation, detection, **and cleanup**. The stale-HANDOFF-triggers-phantom-resume failure mode (visible before Phase 5) is closed; `/gsd:resume-work` step 6 invokes `checkpoint --clear` after successful resume.
4. **Disciplined rescope mid-milestone** — AUDIT-v1.1.md found upstream-compat scope was compromised by upstream drift; Option C trimmed aggressively and shipped. Recorded the decision trail so deferred work (LIFE-02, DOCS-01/02, UPST-01/03/04) has clean v1.2 context.
5. **Parallel wave execution proved out** — both Phase 5 plans (disjoint files, no shared state) ran simultaneously in ~8 minutes. Pattern validated for future small phases.
6. **Plugin-maintenance surface area** — the `/gsd-` → `/gsd:` namespace rewrite surfaced a whole class of upstream-drift issue. Captured as a durable maintenance script + memory rule so post-sync normalization becomes routine.

## Lessons

- **Audit early, rescope aggressively.** The v1.1 mid-milestone audit recovered ~60% of planned scope as deferrable polish. Running that audit before executing the remaining phases would have saved at most a few hours but preserved the same outcome. Consider scheduling milestone audits at phase boundaries when the scope feels speculative.
- **Upstream-coupled phases are fragile.** Phase 6 was defined assuming a specific upstream direction. Between milestone kickoff (2026-04-11) and audit (2026-04-20), upstream shipped ~4 minor versions with orthogonal features. Lesson: avoid scoping phases around "compat with moving upstream" without a specific commit hash to target.
- **Live UAT beats synthetic verification for hook-driven code paths.** Phase 4's synthetic checks all passed, but the live `/compact` test was the only way to confirm the CC hook pipeline actually fires. Future hook-touching phases should front-load the live test.
