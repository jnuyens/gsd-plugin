Create a new file `bin/lib/checkpoint.cjs` that exports two functions:

**`generateCheckpoint(cwd, options)`** -- Gathers all data and returns a HANDOFF.json object (does NOT write to disk).

Parameters:
- `cwd` (string) -- project root directory
- `options` (object):
  - `source` (string, default `"manual-pause"`) -- `"auto-compact"` or `"manual-pause"` per D-01/D-11
  - `contextNotes` (string, optional) -- additional context from caller (pause-work may pass conversation context)
  - `partial` (boolean, default `false`) -- set true if caller hit timeout per D-04

Implementation steps:
1. `require('./core.cjs')` for `planningPaths`, `safeReadFile`, `execGit`, `findPhaseInternal`
2. `require('./frontmatter.cjs')` for `extractFrontmatter`
3. Read STATE.md via `safeReadFile(planningPaths(cwd).state)`
4. Parse STATE.md frontmatter with `extractFrontmatter()` to get `milestone`, `status`, `stopped_at`
5. Extract current position from STATE.md body: parse "Phase:", "Plan:", "Status:" lines using regex patterns like `/Phase:\s*(\d+)/i` and `/Plan:\s*(.*)/i`
6. Run `execGit(cwd, ['status', '--porcelain'])` to get uncommitted files list (per D-03). Split by newlines, trim, filter empty. Each entry is a string like "M  path/to/file" or "?? path/to/file"
7. Run `execGit(cwd, ['log', '--oneline', '-5'])` to get recent commit messages for context
8. Find current phase directory: use `findPhaseInternal(cwd, phaseNumber)` if phase number was parsed from STATE.md
9. If phase dir found, scan for PLAN.md and SUMMARY.md files to build `completed_tasks` and `remaining_tasks` arrays:
   - List files matching `*-PLAN.md` in phase dir
   - For each PLAN.md, check if corresponding `*-SUMMARY.md` exists
   - Plans with SUMMARY = completed (extract plan name from frontmatter or filename)
   - Plans without SUMMARY = remaining
10. Parse STATE.md "Accumulated Context > Decisions" section for `decisions` array
11. Parse STATE.md "Blockers/Concerns" section for `blockers` array
12. Build context_notes string: combine STATE.md "Current Position > Status" + recent git commit subjects (per D-02)
13. Compose and return the HANDOFF.json object matching this exact schema:

```javascript
{
  version: "1.0",
  timestamp: new Date().toISOString(),
  source: options.source || "manual-pause",
  partial: options.partial || false,
  phase: phaseNumber || null,
  phase_name: phaseName || null,
  phase_dir: phaseDir || null,
  plan: currentPlan || null,
  task: currentTask || null,
  total_tasks: totalTasks || null,
  status: options.source === "auto-compact" ? "auto-checkpoint" : "paused",
  completed_tasks: completedTasks,
  remaining_tasks: remainingTasks,
  blockers: blockers,
  human_actions_pending: [],
  decisions: decisions,
  uncommitted_files: uncommittedFiles,
  next_action: nextAction || null,
  context_notes: contextNotes
}
```

**`writeCheckpoint(cwd, options)`** -- Calls `generateCheckpoint()` then writes to `.planning/HANDOFF.json`.

Implementation:
1. Call `generateCheckpoint(cwd, options)` to get the data object
2. Resolve output path: `path.join(planningPaths(cwd).planning, 'HANDOFF.json')`
3. Per D-05: always overwrite if file exists (no check needed, writeFileSync overwrites by default)
4. Per D-06: write to disk only, no git commit
5. Use `fs.writeFileSync(outputPath, JSON.stringify(data, null, 2) + '\n')` for atomic write
6. Return the data object (so callers can use it)

**`cmdCheckpoint(cwd, args, raw)`** -- CLI command handler for `node gsd-tools.cjs checkpoint`.

Implementation:
1. Parse `--source` flag from args (default: `"manual-pause"`)
2. Parse `--context-notes` flag from args (optional string)
3. Call `writeCheckpoint(cwd, { source, contextNotes })`
4. Use `output()` to emit result JSON on stdout
5. Write status message to stderr: `process.stderr.write('GSD: checkpoint saved to .planning/HANDOFF.json\n')`

Export: `module.exports = { generateCheckpoint, writeCheckpoint, cmdCheckpoint };`

Error handling: wrap git operations in try/catch. If git fails, continue with empty arrays for uncommitted_files and set `partial: true` on the result. If STATE.md is missing/empty, still produce a valid HANDOFF.json with null fields. Never throw -- the PreCompact hook has a 5s budget and must not crash.

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-01 | Tampering | HANDOFF.json | accept | File is transient (deleted after resume), lives in .planning/ which is already trusted project state. No secrets stored. |
| T-04-02 | Information Disclosure | HANDOFF.json context_notes | accept | Contains only project state from STATE.md and git commit subjects -- no credentials or PII. File is gitignored by convention. |
| T-04-03 | Denial of Service | checkpoint timeout | mitigate | Wrap git operations in try/catch, produce partial HANDOFF.json on failure (per D-04). Never throw from generateCheckpoint. |
| T-04-04 | Spoofing | HANDOFF.json source field | accept | Source field is informational only (per D-11). Resume treats all HANDOFF.json identically regardless of source value. |
</threat_model>
