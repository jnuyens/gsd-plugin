Enhance the session-start handler within the `case 'hook':` block of gsd-tools.cjs. Add HANDOFF.json detection AFTER the existing auto-migration logic (after the migration try/catch block, before the closing of the session-start branch).

The enhanced session-start block should read stdin to get the `source` field, then conditionally check for HANDOFF.json.

Replace the session-start section (from `if (hookType === 'session-start')` through its closing brace) with:

```javascript
    if (hookType === 'session-start') {
      // Read hook input from stdin for source detection
      let hookInput = {};
      try {
        const stdinData = fs.readFileSync(0, 'utf-8');
        if (stdinData) hookInput = JSON.parse(stdinData);
      } catch { /* stdin may not be available or parseable */ }

      // Auto-migrate legacy artifacts on first session
      try {
        const migrationPath = path.join(__dirname, '..', 'migrations', 'legacy-cleanup.cjs');
        if (fs.existsSync(migrationPath)) {
          const { autoMigrate } = require(migrationPath);
          const result = autoMigrate(cwd);
          if (result.migrated) {
            const msg = ['GSD plugin: auto-migrated legacy install:', ...result.actions.map(a => `  - ${a}`)].join('\n');
            process.stderr.write(msg + '\n');
          }
        }
      } catch { /* never break session start */ }

      // HANDOFF.json detection for auto-resume (D-08)
      // Only check on startup and compact sources
      // Skip on "clear" (user intentionally cleared) and "resume" (avoid loop)
      const source = hookInput.source || 'startup';
      if (source === 'startup' || source === 'compact') {
        try {
          const { planningPaths } = require('./lib/core.cjs');
          const handoffPath = path.join(planningPaths(cwd).planning, 'HANDOFF.json');
          if (fs.existsSync(handoffPath)) {
            // Read HANDOFF.json to include context in the system message
            const handoff = JSON.parse(fs.readFileSync(handoffPath, 'utf-8'));
            const phase = handoff.phase_name || handoff.phase || 'unknown';
            const plan = handoff.plan || '?';
            const task = handoff.task || '?';
            const handoffSource = handoff.source || 'unknown';

            // stdout becomes system message for Claude (per SessionStart hook contract)
            const systemMsg = [
              'GSD session continuity: Found checkpoint from previous session.',
              `Phase: ${phase}, Plan: ${plan}, Task: ${task} (source: ${handoffSource}).`,
              'Run /gsd-resume-work to restore context and continue.',
              'Do this immediately without waiting for user input.'
            ].join(' ');
            process.stdout.write(systemMsg);

            // stderr for user-visible notification
            process.stderr.write('GSD: session checkpoint detected, auto-resuming...\n');
          }
        } catch { /* never break session start */ }
      }
```

Key implementation details:
1. **stdin parsing**: Read stdin JSON to get `source` field. Wrapped in try/catch since stdin may not be available in all contexts.
2. **Source filtering (per research section 4)**:
   - `startup`: Check for HANDOFF.json -- this is a fresh session, resume if checkpoint exists
   - `compact`: Check for HANDOFF.json -- we just wrote it during PreCompact, session needs to restore context
   - `clear`: Skip -- user intentionally cleared context. If they want to resume, they'll run /gsd-resume-work manually
   - `resume`: Skip -- already in a resume flow, avoid infinite loop
3. **stdout = system message**: Per the hook contract (executeHooks generator), stdout content becomes a system message that Claude sees as conversation context. This is how we tell Claude to run /gsd-resume-work without user intervention (D-08).
4. **System message content**: Directive and concise. Includes phase/plan/task context so Claude has immediate orientation. Ends with "Do this immediately" to ensure zero-intervention behavior (D-09, RESM-02).
5. **stderr = user notification**: User sees "auto-resuming..." in their terminal so they know what's happening.
6. **Error safety**: Entire HANDOFF.json detection wrapped in try/catch. If anything fails, session starts normally without resume -- user can always run /gsd-resume-work manually.

IMPORTANT: The pre-compact handler added by Plan 02 must remain intact. Only modify the session-start section.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| SessionStart stdin | Claude Code runtime provides session metadata as JSON |
| SessionStart stdout | Output injected as system message into conversation |
| HANDOFF.json content | Read from disk, parsed as JSON, included in system message |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-08 | Spoofing | HANDOFF.json triggering unwanted resume | mitigate | Source filtering: skip on "clear" and "resume" to prevent loops. HANDOFF.json is deleted after successful resume by resume-work skill (LIFE-01). |
| T-04-09 | Information Disclosure | HANDOFF.json content in system message | accept | System message only includes phase name, plan number, task number, and source -- no secrets. Full HANDOFF.json content is read by resume-work skill, not injected into system prompt. |
| T-04-10 | Denial of Service | Infinite resume loop | mitigate | Source "resume" is explicitly skipped. resume-work deletes HANDOFF.json after successful resume. If deletion fails, next startup triggers resume again which is acceptable (retry behavior). |
| T-04-11 | Tampering | Malicious HANDOFF.json | accept | HANDOFF.json is written by our own checkpoint code to .planning/ directory. Attacker with write access to .planning/ already has full project access. JSON.parse failure caught by try/catch. |
</threat_model>
