**Shipped:** 2026-04-21 (final polish landing 2026-04-24)
**Duration:** 2026-04-20 → 2026-04-24 (5 days, most of the work in ~26 minutes of executor time)

## Overview

v1.1 closed the session-continuity feature but exposed a meta-problem: the plugin sits downstream of a rapidly-moving upstream ([GSD](https://github.com/gsd-build/get-shit-done) shipped 1.34 → 1.38.x during v1.1), and drift was caught ad-hoc by humans noticing bugs. v1.2 made drift detection a first-class plugin concern. Three detectors now run in CI on every push — file-layout, HANDOFF schema, and namespace — each with a committed ratchet baseline. An umbrella orchestrator + a post-sync upstream-schema drift detector close the loop at sync time.

The milestone also absorbed the reframed v1.1 UPST-01 requirement (originally "make HANDOFF compatible for upstream PR"; reframed to "detect drift between upstream's `pause-work` output and our schema"). Phase 8's research confirmed upstream's schema is stable across inspected versions (1.37.1, 1.38.1, 1.38.3) and that the plugin is a strict superset of upstream's 17-field contract.

### Phase 7: File-Layout Drift Detector
**Goal:** Plugin-local script catches dangling `@~/.claude/...` references in skills/agents/references before they ship. CI enforces via ratchet baseline.
**Depends on:** Nothing new (runs against the current plugin layout).
**Requirements:** DRIFT-01, DRIFT-02 (file-layout portion)
**Success Criteria (met):**
  1. Detector script scans all SKILL.md, agent.md, reference, template, and bin/ files for `@~/.claude/...` and `@$HOME/.claude/...` path references ✓
  2. Classifies each: Category A (plugin has local counterpart) vs Category B (genuinely missing) ✓
  3. Compares counts to `tests/drift-baseline.json`; exits non-zero on regression beyond baseline ✓
  4. CI invokes the detector via `.github/workflows/check-drift.yml`; a failing detector blocks merge ✓
  5. The resume-project.md gap from Phase 5 would have been caught retroactively (verified via manual injection + removal) ✓

**Plans:** 1 plan (4 tasks)
- [x] `07-01-PLAN.md` — Detector script, baseline file, first CI workflow, README documentation + smoke test

**Baseline at ship:** 109 total dangling / 38 repairable / 71 genuinely missing. All 71 missing refs are in `workflows/*.md` (structural drift that requires a future decision: inline content into skills, create a plugin-local `workflows/` dir, or drop the references).

### Phase 8: HANDOFF Schema Baseline and Detector
**Goal:** Committed JSON Schema defines the HANDOFF contract; plugin output validates in CI; post-sync check catches upstream drift.
**Depends on:** Phase 7 (detector pattern established); research R-1 (upstream `pause-work` reconnaissance — resolved inline at plan time).
**Requirements:** SCHEMA-01, SCHEMA-02, SCHEMA-03, DRIFT-02 (schema portion)
**Success Criteria (met):**
  1. `schema/handoff-v1.json` exists, JSON Schema draft-07, describes the 19-field contract (17 required upstream-compat fields + 2 optional plugin extensions `source` and `partial`) ✓
  2. CI validates `writeCheckpoint()` output against the schema on every commit (`handoff-schema` job) ✓
  3. `check-upstream-schema.cjs` reads upstream's pause-work declared field set and diffs against schema; exits non-zero on drift ✓
  4. Detected drift produces a structured diff report (added fields / missing required fields / type changes) ✓
  5. R-1 answered: upstream schema is stable across 1.37.x → 1.38.x (byte-identical workflow body in inspected versions); plugin is a strict superset ✓

**Plans:** 1 plan (6 tasks)
- [x] `08-01-PLAN.md` — ajv + ajv-formats devDependencies, schema + fixture, schema validator, upstream drift detector, CI extension, README + PROJECT.md updates

**Schema summary:**
- 17 upstream-compat required fields: `version, timestamp, phase, phase_name, phase_dir, plan, task, total_tasks, status, completed_tasks, remaining_tasks, blockers, human_actions_pending, decisions, uncommitted_files, next_action, context_notes`
- 2 plugin-only optional fields: `source` (auto-compact vs manual-pause), `partial` (5s-budget timeout flag)
- `additionalProperties: true` — future extensions can land without schema churn
- Research R-1 deliverable at `v1.2-phases/08-handoff-schema-detector/08-RESEARCH.md`

**Plans:** 1 plan (4 tasks)
- [x] `09-01-PLAN.md` — Umbrella orchestrator, README feature tour, CHANGELOG scaffold, PROJECT.md post-sync checklist update

**Key design calls:**
- Umbrella uses `spawnSync` with `stdio: 'inherit'` — pipes child output verbatim, never re-summarizes.
- `check-upstream-schema.cjs` deliberately excluded from umbrella (network-dependent + post-sync-only).
- Umbrella NOT added as a new CI job — CI runs each detector as its own parallel job for fast-feedback granularity.
- Namespace detection via `rewrite-command-namespace.cjs --dry` output parsing (counts "Total replacements: N" — N > 0 = drift).

## Key accomplishments

1. **Three-detector drift-catch in CI** — file-layout, HANDOFF schema, and namespace drift each fail the build if regressed. Ratchet baselines let the plugin ship with known debt (e.g., the 71 Category-B file-layout refs) while preventing further regressions.
2. **Upstream schema research resolved the UPST-01 reframe** — plugin is a strict superset of upstream; schema-drift detector confirmed clean against 1.38.3. Upstream-compat question is settled for now; any future upstream schema change fires the detector at sync time.
3. **Unified umbrella orchestrator** — `check-drift.cjs` gives local dev + post-sync a single entry point while CI stays per-category for fast-feedback granularity.
4. **First CI on the repo** — Phase 7 bootstrapped `.github/workflows/check-drift.yml`; Phase 8 extended it with a second parallel job. Sets the pattern for any future detector-style work.
5. **Post-sync workflow formalization** — PROJECT.md's "After each upstream GSD sync" block went from a memory-rule to a 9-step checklist with detector-pass gates.
6. **Plugin-structure alignment with upstream** — the 260424-srn skill-dir rename (`skills/gsd-<name>/` → `skills/<name>/`) fixed a tab-completion UX bug AND mirrors upstream's `commands/gsd/<name>.md` layout. Future sync friction reduced.

## Lessons

- **Run the actual detector in dry mode before writing plan acceptance criteria.** Phase 7's baseline counts in the plan (107/37/70) were derived from a scoped grep; the real detector scope landed at 112/38/74. Executor had to deviate and commit the measured values. Pre-planning dry-run would have caught this.
- **Executor deviations are signal, not noise.** All three phases had the executor catch and fix plan bugs (regex greediness, cached-path handling, self-colliding regression-test strings). Good instincts, worth preserving in future agent briefs.
- **Research-at-plan-time is cheap when the answer is in cached artifacts.** R-1 resolved in ~10 min using the upstream tarballs already extracted from the morning's sync. A dedicated research agent would have duplicated effort.
- **Detector-pattern reuse pays.** Phase 7's script skeleton (`use strict`, env guard, git ls-files walk, exit 0/1/2 convention, `--dry` flag) was lifted into Phase 8's two new detectors and Phase 9's umbrella without ceremony.
